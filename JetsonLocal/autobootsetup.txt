AURA Jetson Auto-Boot Steps

1. Put your files in the JetsonLocal folder
- nano_main.py
- static/index.html
- static/script.js
- static/style.css
- requirements.txt
- setup_orin.sh
- launch_kiosk.sh
- aura-edge.service
- aura-kiosk.desktop

2. Edit your .env file
Use the right serial port for the ESP32:
SERIAL_PORT=/dev/ttyACM0

You can also add:
AURA_LOCAL_APP_HOST=127.0.0.1
AURA_LOCAL_APP_PORT=8000
AURA_OLLAMA_URL=http://127.0.0.1:11434
LOCAL_DB_NAME=jetson_local_db

3. Run setup
From JetsonLocal:
chmod +x setup_orin.sh
./setup_orin.sh

4. Test the local app manually
source aura_env/bin/activate
python nano_main.py

Then open:
http://127.0.0.1:8000

5. Test serial manually
Make sure the ESP32 is on the expected port:
ls /dev/ttyACM*
or
ls /dev/ttyUSB*

If needed update SERIAL_PORT in .env.

6. Make kiosk launcher executable
chmod +x launch_kiosk.sh

7. Edit the service file
Replace YOUR_USERNAME in aura-edge.service with your real Linux username.

8. Install the systemd service
sudo cp aura-edge.service /etc/systemd/system/aura-edge.service
sudo systemctl daemon-reload
sudo systemctl enable aura-edge.service
sudo systemctl start aura-edge.service

9. Check the service
sudo systemctl status aura-edge.service

10. Edit desktop autostart file
Replace YOUR_USERNAME in aura-kiosk.desktop with your real Linux username.

11. Install desktop autostart
mkdir -p ~/.config/autostart
cp aura-kiosk.desktop ~/.config/autostart/aura-kiosk.desktop

12. Reboot test
sudo reboot

13. After reboot, expected behavior
- Ollama starts
- AURA local app starts on 127.0.0.1:8000
- desktop login opens the kiosk app
- screen rotates right
- local touchscreen app fills the screen

14. If rotation command does not work
Try:
xrandr --query

Then replace the rotate line in launch_kiosk.sh with something like:
xrandr --output HDMI-0 --rotate right

15. Useful checks
Service logs:
journalctl -u aura-edge.service -n 100 --no-pager

Check app directly:
curl http://127.0.0.1:8000

Check Ollama:
ollama list

16. If serial does not connect
Check permissions:
groups

Sometimes you need to add your user to dialout:
sudo usermod -aG dialout $USER

Then log out and back in.
